# 狼人杀游戏桌布局优化

## 🎯 功能概述

将原来的圆形玩家布局改为更符合真实狼人杀游戏场景的**两侧对坐布局**，每侧6个槽位，总计12个玩家位置。

## 📐 布局设计

### 整体结构

```
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│  左侧玩家   │         │  中央游戏桌  │         │  右侧玩家   │
│  (1-6号位)  │ ◄─────► │    (圆桌)    │ ◄─────► │  (7-12号位) │
└─────────────┘         └─────────────┘         └─────────────┘
```

### 左侧玩家区 (1-6号位)
- 垂直排列，从上到下
- 每个槽位显示玩家头像、姓名、角色
- 存活玩家：绿色边框 + 呼吸灯效果
- 已淘汰玩家：灰色 + 半透明
- 空位：虚线边框 + "空位"占位符

### 中央游戏桌
- 圆形渐变背景
- 缓慢旋转动画（60秒一圈）
- 虚线边框反向旋转
- 显示内容：
  - 当前发言玩家（头像 + 姓名 + "正在发言..."）
  - 轮次信息（第X轮）
  - 阶段信息（讨论/投票）
  - 游戏提示

### 右侧玩家区 (7-12号位)
- 与左侧对称
- 相同的样式和交互效果

## 🎨 视觉效果

### 玩家卡片状态

#### 1. 存活玩家 (slot-alive)
- 背景：绿色渐变 (#e8f5e9 → #a5d6a7)
- 边框：绿色 (#4caf50)
- 动画：呼吸灯效果（aliveGlow）
- 悬停：向外移动5px + 放大1.02倍

#### 2. 已淘汰玩家 (slot-dead)
- 背景：灰褐色渐变
- 边框：棕色 (#795548)
- 透明度：60%
- 无动画效果

#### 3. 正在发言 (slot-speaking)
- 背景：蓝色渐变 (#e3f2fd → #90caf9)
- 边框：紫色 (#667eea)
- 动画：脉冲效果 + 发光阴影
- 头像：上下跳动（iconBounce）

#### 4. 空槽位 (slot-empty)
- 背景：浅灰渐变
- 边框：虚线灰色
- 透明度：40%
- 显示槽位编号 + "空位"文字

### 特殊效果

#### 中央游戏桌动画
- 主体：缓慢顺时针旋转（60s）
- 边框：快速逆时针旋转（20s）
- 夜晚模式：切换为深色主题

#### 玩家交互
- 悬停：显示详细信息卡片（Tooltip）
- 信息包括：姓名、角色图标、存活状态
- 淡入动画 + 位置过渡

## 📱 响应式适配

### 大屏幕 (> 1400px)
- 标准三栏布局
- 左右各220px，中间350px

### 中等屏幕 (1200px - 1400px)
- 缩小尺寸，保持三栏布局
- 左右各180px，中间280px

### 小屏幕 (< 1200px)
- 改为垂直布局：
  - 中央游戏桌在最上方
  - 左侧玩家横向排列
  - 右侧玩家横向排列
- 自动换行适应宽度

### 移动端 (< 768px)
- 进一步缩小卡片尺寸
- 头像图标2rem
- 游戏桌200x200px

## 🎮 功能特性

### 槽位管理
- `getPlayerBySlotNum(slotNum)`: 根据槽位号获取玩家
- `getSlotClass(slotNum)`: 获取槽位状态类名
- `getPlayerIcon(player)`: 根据角色返回对应emoji

### 角色头像映射
```javascript
🐺 - 狼人
👨 - 村民
🔮 - 预言家
🧙 - 女巫
🏹 - 猎人
💀 - 已淘汰
```

### 状态同步
- 实时更新玩家存活状态
- 高亮当前发言者
- 自动显示空槽位

## 💡 使用建议

### 创建游戏
1. 建议选择 **6-12人** 游戏模式
2. 少于6人时，部分槽位显示为空位
3. 大于12人时，需要扩展槽位（未来功能）

### 观看体验
- **左右对比**: 清晰看到两阵营对决
- **中央聚焦**: 关注当前发言者
- **悬停查看**: 查看详细玩家信息

## 🔄 与流式输出的整合

该布局完美配合流式对话功能：

1. **发言时**：玩家卡片高亮 + 脉冲效果
2. **头像动画**：上下跳动吸引注意
3. **中央显示**：同步显示发言者信息
4. **事件日志**：下方滚动显示完整对话
